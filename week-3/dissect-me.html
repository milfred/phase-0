The Benefits of Using Git and GitHub

Why a Version Control System is Essential to Modern Software Development

February 13, 2016

I've only been a budding developer for a little less than a week now, but one thing I can say for certain is that version control and GitHub will be an important part of my life as I progress as a programmer. A version control system (VCS) is a way to track changes to software and a means for developers to work on the same project simultaneously, while avoiding conflicts or overwriting each other's code.

The key benefits to using a VCS are:

  • Orginization. Every time a set of changes is commited, it is logged and a version of the repository is saved. Team members can see the entire history of changes to the project and view messages regarding what changes were made and when. It also eliminates the need to manage, save, and name mutiple versions of your files on your own.
  • Teamwork. With a VCS, every member of the team working on a project can work without restrictions or fear of overwriting a file someone else is working on because every memeber has a full version or the repository on their computer. Adding messages to each commit helps your teammates know what work has been done.
  • Security. Because a version of the repository is saved after each commit, the project can be rolled back to an earlier version if something goes wrong. There's no need to worry about hardware failure, accidental damage, or inadvertant overwriting of data when you use a VCS. All you need to do to recover what you lost is pull the repository from one of your contributors. With services like GitHub, you can also store a master version of the repository in the cloud, further protecting your data.
  • Git is a an Open Source VCS that helps keep track of the file changes for a project. It's basically a filing system that saves a version of every draft of file. It does this by viewing all files in three ways. Modified/Untracked, Staged, and Committed. While you are working on a file, it is in the modified state. Once you are done changing it, you can add it to the staging area to be committed once you've finished modifing all the files you want for that change/commit. Once you commit those files in the staging area, you have essentially created a new snapshot or save point of your repository that you can refer back to or completely roll back your project to.

GitHub is one part cloud storage for you git repositories and one part social network for developers. It hosts a central repository for your project that can be pulled down to your local machine for you to work on. Then you push the modified repository back up to the central repo when you're done. All of the contributors to a project can view each version change and what was updated. As stated above, having a central repository hosted on GitHub adds an extra measure of protection against accidental data loss.
